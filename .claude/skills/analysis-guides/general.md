# 汎用データ分析ガイドライン

domain-expertエージェントが参照する、分析対象の分野（EC、金融等）によらない共通の分析ガイドライン。

## データ品質チェックリスト

- [ ] 欠損値のパターン（ランダムか構造的か）
- [ ] 重複行の有無と原因
- [ ] 外れ値の確認（ビジネス的に妥当か）
- [ ] データ型の妥当性（数値なのに文字列になっていないか）
- [ ] IDカラムのユニーク性
- [ ] 日付の範囲と連続性
- [ ] カテゴリ変数の値の妥当性

## 分析の優先順位

1. **データ全体像の把握** - shape, 型, 欠損値, 分布
2. **ターゲット変数の理解** - 分布, 偏り, 欠損
3. **特徴量とターゲットの関係** - 相関, 条件付き分布
4. **特徴量間の関係** - 多重共線性, 交互作用
5. **時間的パターン** - トレンド, 季節性, 変化点

## よくあるデータの落とし穴

| 問題 | 兆候 | 対処 |
|---|---|---|
| データリーク | ターゲットと異常に高い相関 | 時系列的に未来の情報が含まれていないか確認 |
| 選択バイアス | 特定条件のデータのみ | データ収集プロセスを確認 |
| 生存者バイアス | 成功事例のみのデータ | 脱落したデータがないか確認 |
| クラス不均衡 | ターゲットの分布が極端 | リサンプリングや評価指標の工夫 |
| 多重共線性 | 複数カラムが高相関 | VIFの確認、片方の除外を検討 |

## 統計量の解釈ガイド

### 相関係数
- |r| > 0.8: 強い相関 → 多重共線性に注意
- 0.5 < |r| < 0.8: 中程度の相関 → 有用な特徴量候補
- |r| < 0.3: 弱い相関 → 非線形関係の可能性も検討

### 欠損値
- < 5%: 通常は補完で対処可能
- 5-30%: 補完方法の慎重な選択が必要
- > 30%: カラムの除外を検討

### カーディナリティ
- ユニーク率 > 95%: IDカラムの可能性（特徴量としては不適切）
- ユニーク値 = 1: 定数カラム（情報なし、除外）
- ユニーク値 = 2: バイナリ変数
