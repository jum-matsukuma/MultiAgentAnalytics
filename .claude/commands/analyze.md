# Quick Data Analysis

指定されたデータファイルに対して、包括的なクイック分析を実行し、レポートを生成する。

## Usage

- `/analyze <file>` - データファイルを分析
- `/analyze <directory>` - ディレクトリ内のデータファイルをすべて分析
- `/analyze <file1> <file2> ...` - 複数ファイルを関連付けて分析

## 引数

`$ARGUMENTS` にはデータファイル、ディレクトリ、またはスペース区切りの複数パスを指定する（例: `data/sales.csv`, `data/`, `data/train.csv data/test.csv`）。

## 前処理: 入力の判定

**スペース区切りで複数パスが指定された場合:**
1. 各パスのデータファイルを列挙する（ディレクトリならその中のデータファイルを展開）
2. 出力先は `output/` とする
3. 各ファイルを個別に分析した後、**ファイル間の横断分析**を行う
4. 共通カラム（キー）の特定、スキーマ比較、データの関係性（train/test分割、時系列の連続性など）を分析する
5. 最後に全ファイルの関係性を含む統合サマリを `output/summary.md` に出力する

**単一のディレクトリが指定された場合:** その中のデータファイル（`.csv`, `.tsv`, `.parquet`, `.json`）を列挙し、上記の複数ファイル分析と同様に処理する。出力先は `output/<ディレクトリ名>/` とする。

**単一のファイルが指定された場合:** 出力先は `output/` とする。

以下では対象ファイルを `<file>`、出力先を `<outdir>` と表記する。

## 実行手順

以下の手順を**すべて自動で**実行し、最終的に1つのMarkdownレポートを出力すること。

### 1. データ概要の把握

```bash
uv run edatool summarize <file>
```

結果を確認し、データの行数・列数・型・基本統計を把握する。

### 2. データ品質チェック

```bash
uv run edatool quality-check <file>
```

欠損値・重複・カーディナリティの問題を確認する。

### 3. 主要カラムのヒストグラム生成

数値カラムの中から重要と思われるもの（最大5つ）を選び、ヒストグラムを生成する:

```bash
uv run edatool plot histogram <file> --column <col> -o <outdir>/plot_hist_<col>.png
```

カテゴリカルカラムで重要なものがあればそちらも含める。

### 4. レポート生成

上記の結果を統合し、以下の構成でMarkdownレポートを `<outdir>/report_quick_analysis.md` に保存する:

```markdown
# クイック分析レポート: <ファイル名>

## データ概要
- 行数・列数・型の概要

## データ品質
- 品質チェック結果のサマリ
- 問題点があれば明記

## 主要カラムの分布
- 各ヒストグラムの画像を埋め込み（`![](plot_hist_<col>.png)`）
- 分布の特徴を簡潔にコメント

## 所見
- データの特徴や気づいた点を箇条書き
- 深掘りすべきポイントの提案
```

## 注意事項

- 出力ディレクトリがなければ作成すること
- レポート内の画像パスは相対パスで記載する
- 全工程を自動で実行し、ユーザーへの確認は不要
- 複数ファイル指定時は、ファイル名や中身から関係性を推測し（train/test分割、マスタ/トランザクション、時系列の連続性など）、**最初からその関係を前提にした一体的な分析**を行う。関係が明らかでない場合のみ個別に分析する
- 最後にレポートのパスをユーザーに伝える
