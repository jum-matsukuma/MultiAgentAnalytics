# Deep Data Analysis

指定されたデータファイルに対して、マルチエージェントによる包括的な深層分析を実行する。

## Usage

- `/analyze-deep <file>` - データファイルを深層分析
- `/analyze-deep <directory>` - ディレクトリ内のデータファイルをすべて深層分析
- `/analyze-deep <file1> <file2> ...` - 複数ファイルを関連付けて深層分析

## 引数

`$ARGUMENTS` にはデータファイル、ディレクトリ、またはスペース区切りの複数パスを指定する（例: `data/sales.csv`, `data/`, `data/train.csv data/test.csv`）。

## 前処理: 入力の判定

**スペース区切りで複数パスが指定された場合:**
1. 各パスのデータファイルを列挙する（ディレクトリならその中のデータファイルを展開）
2. 出力先は `output/` とする
3. ファイル名や中身から関係性を推測し（train/test分割、マスタ/トランザクション、時系列の連続性など）、**最初からその関係を前提にした一体的な分析**を行う
4. 関係が明らかでない場合は各ファイルを個別に分析し、最後にサマリで比較する
5. 全体サマリを `output/summary.md` に出力する

**単一のディレクトリが指定された場合:** その中のデータファイル（`.csv`, `.tsv`, `.parquet`, `.json`）を列挙し、上記の複数ファイル分析と同様に処理する。出力ルートは `output/<ディレクトリ名>/` とする。

**単一のファイルが指定された場合:** 出力先は `output/` とする。

以下では対象ファイルを `<file>`、各ファイルの出力先を `<outdir>` と表記する。

## 実行手順

以下の手順を**すべて自動で**実行し、最終的に詳細なMarkdownレポートを出力すること。

### Phase 1: ドメイン理解と方針策定

domain-expert エージェントを起動し、データの概要からドメインを特定・分析方針を助言させる:

```
Agent(subagent_type="domain-expert"):
  データファイル: <file>（複数ファイルの場合は全ファイル）
  1. uv run edatool summarize <file> を各ファイルに実行してデータ概要を確認
  2. ドメインを特定し、注目すべきカラム・分析の方向性を提案
  3. 複数ファイルの場合はファイル間の関係性を判断し（train/test分割、マスタ/トランザクション等）、
     一体的に分析するか個別に分析するかの方針を決定する
  4. 結果を <outdir>/advice_direction.md に保存
```

### Phase 2: 詳細分析

data-analyst エージェントを起動し、フルプロファイルと深掘り分析を実行させる:

```
Agent(subagent_type="data-analyst"):
  データファイル: <file>（複数ファイルの場合は全ファイル）
  Phase 1のアドバイス（<outdir>/advice_direction.md）を読み、それを踏まえて:
  1. uv run edatool profile <file> を各ファイルに実行（stats + correlations + quality check を含む）
  2. ターゲット変数が特定できる場合のみ uv run edatool correlations <file> --target <col> を追加実行
  3. アドバイスで指摘された追加分析があれば Python API で実施
  4. 複数ファイルの場合はファイル間のスキーマ比較・共通キーの特定・データ整合性チェックも実施
  5. 結果を <outdir>/analysis_detailed.md に保存
```

### Phase 3: 可視化

visualizer エージェントを起動し、Phase 1-2 の結果に基づいて重要な可視化を生成させる:

```
Agent(subagent_type="visualizer"):
  データファイル: <file>（複数ファイルの場合は全ファイル）
  <outdir>/advice_direction.md を読み（<outdir>/analysis_detailed.md が存在すればそちらも参照）:
  1. 重要な数値カラムのヒストグラム（最大5つ）
  2. 重要な変数ペアの散布図（最大3つ）
  3. 相関ヒートマップ
  4. ドメイン固有で有用なグラフがあれば追加
  5. 複数ファイルの場合はファイル間の分布比較グラフも生成
  すべて <outdir>/ に保存（命名規則: plot_<type>_<desc>.png）
```

### Phase 4: レポート統合

reporter エージェントを起動し、全成果物を統合したレポートを作成させる:

```
Agent(subagent_type="reporter"):
  <outdir>/ 内の全成果物を読み込み、以下の構成でレポートを作成:

  # 深層分析レポート: <ファイル名>

  ## エグゼクティブサマリ
  ## ドメインコンテキスト
  ## データ概要
  ## データ品質
  ## 統計分析
  ## 相関分析
  ## 可視化
  ## 詳細な所見と推奨事項

  レポートを <outdir>/report_deep_analysis.md に保存
```

### Phase 5: 全体サマリの生成（複数ファイル/ディレクトリ指定時）

全ファイルの分析完了後、以下の構成で `output/<ディレクトリ名>/summary.md` を作成する:

```markdown
# データセットサマリ: <ディレクトリ名>

## ファイル一覧
- 各ファイルの行数・列数・概要を表形式で

## ファイル間の関係性
- 共通カラム（キー）の特定
- train/test/validation等の分割構造
- ファイル間の参照関係（IDの対応など）

## データセット全体の特徴
- ドメイン・タスクの推定
- 全体を通じた品質上の注意点

## 深掘りの推奨事項
- 次に確認すべきポイント
```

## 注意事項

- 出力ディレクトリがなければ作成すること
- Phase 1 → 2・3（並列可）→ 4 の順に実行する
- Phase 2 と Phase 3 は Phase 1 完了後に並列実行してよい（Phase 3 は advice_direction.md があれば開始可能。analysis_detailed.md は存在すれば参照する）
- 各エージェントの中間成果物も出力ディレクトリに残す
- 複数ファイル指定時は、ファイル名や中身から関係性を推測し、**最初からその関係を前提にした一体的な分析**を行う。Phase 1 の domain-expert がファイル間の関係性を判断し、以降の Phase に方針を伝える
- 最後にレポートのパス（複数ファイル/ディレクトリ指定時はsummary.mdも含む）をユーザーに伝える
- 全工程を自動で実行し、ユーザーへの確認は不要
